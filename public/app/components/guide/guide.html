Alexa Skill Setup Guide

This guide serves as a simplified walkthrough for setting up an Alexa Skill.

This guide does not provide guidelines to set up an Alexa Smart Home Skill


There are several ways to configure Alexa Skills. We at elev8 have curated a simplified step-by-step process that any person, with limited technical knowledge, can follow to configure the skill.


Glossary
  • Alexa Skill : Alexa’s capabilities to perform tasks. Skills enable customers to verbally interact with devices in a more intuitive way. Examples of skills include the ability to play music, answer general questions, set an alarm or timer, etc...
  • Alexa Skills Kit : a software framework that allows developers to add skills to Alexa.

    See : Alexa Skills Kit

  • Custom Alexa Skill : a skill that can be custom-built and personalized with specific user requirements
  • Alexa Smart Home Skill : an Alexa Skill that is specialized in controlling connected IOT (Internet of Things) devices.

    See : Smart Home Skills

  • (Amazon Web Services) : cloud-based service by Amazon where Alexa code runs.
  • Lambda Functions : The Alexa code which runs on AWS.

    See : AWS Lambda


Note: If you do not Alexa-enabled device, you can use third party Alexa-enabled services, such as Echosim.io to test your Alexa Skill.

Setup

Setting up an Alexa Skill consists of 2 major steps :

  1. Setting up the brain of the Alexa Skill (i.e Lambda Functions)
  2. Registering the Alexa Skill

A. Setting up the brain of the Alexa Skill.

see: Creating an AWS Lambda Function for a Skill for more details.

  1. Sign in or Create an Account in Amazon Web Services

    A free account can handle an Alexa Skill on almost every occasion. Check out Lambda free tier for more info.

  2. Log on to the AWS Management Console and navigate to AWS Lambda.
  3. Click the region drop-down in the upper-right corner of the console window and select US East (N. Virginia).
  4. Click the ‘Lambda’ link under the Compute section (fig. 1).

  5. Click 'Create a Lambda Function' → Select the blueprint tab.
  6. In the blueprint tab, click ‘Skip’ at the bottom right.
  7. When prompted to configure triggers (fig. 2), click the outlined box and select Alexa Skills Kit, then click 'Next' which will show the Configure Function View.

  8. Configure Function View – change the items listed below:
    1. Give a Name to the function.
    2. Give a Description to the function.
    3. Keep the runtime as Node.js 4.3
    4. For Code entry type – select option as ‘Upload a .ZIP file’ and upload the provided Alexa skill zip file.
    5. For Role, select the option 'Create new role' which opens IAM console.

      If you have created a role before, select 'Use Existing role'.

    6. In the IAM console, change the IAM Role option to 'Create a new IAM Role' , enter a Role Name, and click 'Allow'.
    7. Click ‘Next’ on the bottom right of the view which switches to Review View.
  9. Review View – click the ‘Create function’ button → opens up Alexa Skill Details View.
  10. You have successfully created a lambda function!
  11. Alexa Skill Details View – copy the ARN value on the top right of the view as it will be needed in the proceeding steps.

B. Registering the Alexa Skill.

see: Registering an Alexa Skill for more details

  1. Create an account or sign in to your Amazon Developer account.
  2. After signing in, click the ‘Alexa’ tab on the web page.
  3. In the next window, click on the Alexa Skills Kit button (fig. 3).

  4. Click the ‘Add a New Skill’ button on the top right of the next window.
  5. Fill in the necessary information for the ‘Skill Information’.
    1. Name - provide the name you want your Alexa Skill to have (this is the name that would show up in the Alexa Skills Library).
    2. Invocation Name - the name that is used to activate your app.
    3. Click ‘Next’ which opens the Interaction Model section.
  6. Interaction Model section
    1. Unzip the previously downloaded Alexa Skill zip file and locate the files 'intent_schema.json' and 'intent_utterances'
    2. In the Intent Schema section, paste the contents in the provided file named ‘intent_schema.json’
    3. In the Sample Utterances section, paste the contents in the provided file named ‘intent_utterances’
    4. Click ‘Next’ which will take you to the Configuration section.
  7. Configuration section
    1. Select the Service Endpoint Type as ‘AWS Lambda ARN (Amazon Resource Name)’
    2. Select the region for ‘AWS Lambda ARN’ as ‘North America’
    3. Paste the ARN value which was copied from the Amazon AWS website (fig. 4).

    4. Then click ‘Next’ which will switch you to the Test tab.
  8. Test Tab - you can test the skill here if you want to.

    See: Testing a Custom Skill for more information.

    1. You can skip this stage by clicking ‘Next’ which opens the Publishing Information tab.
  9. Publish Information tab

    See : Policy Guidelines for more information.

    1. Click ‘Next’ after filling all the required information. This will lead you to the Privacy and Compliance tab
  10. Privacy and Compliance Tab
    1. After providing the required data, click the ‘Submit for Certification’ button.

      See: Submitting the Skill for Certification

Congratulations! You have successfully set up an Alexa Skill!
  • After several days, you will receive a notification from Amazon via email mentioning that your skill has been certified and the date your skill will be available on the Amazon Alexa Skills Library.

    see : Publication Status for more details.

  • Once you receive the message, install the skill on your Alexa app in order to start using the skill on your Amazon Alexa device.
  • If the skill could not be certified, an email will be sent by Amazon with information on what needs to be corrected.

    see: Certification Requirements for Custom Skills for more details.

  • Pay close attention and adhere to the guidelines when entering publishing information. Failure to follow the guidelines will cause Amazon to deny certification to the Alexa Skill.
